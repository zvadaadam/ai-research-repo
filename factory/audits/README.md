# Factory Audits

Use this folder for periodic checks that the repo is still navigable.

Useful audit questions:

- Can the current thesis be found in under one minute?
- Is `RESEARCH.md` still a dashboard, not a diary?
- Are claims compressed, or have they become run ledgers?
- Are there stale `running` experiments?
- Has any branch passed five serious runs without synthesis?
- Are we repeating a failed mechanism under a new name?

Audits should end with one of:

```text
continue
stop
pivot
compress before more experiments
```
